### PR #8 — fix: withSentryConfig wrap sempre quando DSN setado (`157c84f`)

Bug em PR #7: condicional pra `withSentryConfig` só wrappava quando `SENTRY_ORG + SENTRY_PROJECT + SENTRY_AUTH_TOKEN` setados (envs de source maps). Sem o wrap, webpack/turbopack instrumentation do Sentry não acontece — SDK fica em estado parcial.

**Lição:** `withSentryConfig` faz muito mais que source maps. É **obrigatório** pro SDK runtime. Source maps são opcionais dentro do wrap.

Comprovado pós-fix: **2 transactions accepted** no Sentry dashboard via CLI.
