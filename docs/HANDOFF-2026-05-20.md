### PR #10 — fix Drizzle Date sql template (`88b18cc`)

`src/lib/db/queries/meta-connections.ts:200` — `listConnectionsNeedingRefresh()` usava:

```ts
// ❌ ERRADO — Date serializa com toString() (formato JS)
sql`${metaConnections.tokenExpiresAt} < ${threshold}`
```

Postgres pooler em transaction mode rejeita parse de "Wed May 27 2026 GMT+0000". Drizzle wrappa erro em `Failed query: ...` sem expor causa real.

**Fix**:

```ts
import { lt } from 'drizzle-orm'
lt(metaConnections.tokenExpiresAt, threshold)
```

Drizzle serializa via column type (timestamptz → ISO). Lição documentada em `~/.claude/projects/.../memory/reference_drizzle_date_sql_template_gotcha.md`.

## Aprendizados acumulados em memória (3 gotchas reusáveis)

1. **`vercel.json` headers gotcha** — bloco `headers` em vercel.json sobrescreve next.config no edge. `pnpm dev` não processa vercel.json. Validar curl em prod URL.
2. **Sentry Next.js setup gotcha** — `withSentryConfig` wrap obrigatório pro runtime SDK funcionar, não só source maps. Condicional pra source maps apenas dentro do wrap.
3. **Drizzle Date sql template gotcha** (este audit) — Drizzle `sql\`${col} < ${date}\``serializa Date com`toString()`, não ISO. Postgres pooler rejeita silenciosamente. Sempre usar operadores nativos (`lt/gt/eq/between`) com colunas timestamp.
