## Fluxo do `analyze(prompt, input, ctx, options?)`

1. **Pré-flight de créditos** (se `ctx.credits`): `checkBalance(ws, cost, {safetyFactor:1.5})` → bloqueia saldo limítrofe (§4.11). Sem créditos → pula.
2. **runOnce**: **budget gate** (Regra 20, soma `claude_request_logs.cost_usd` do mês) → request com **prompt caching** (`cache_control: ephemeral` no system) + **retry** 429/5xx (backoff 10s/30s/2min, sem retry 4xx) → `validateOutput` Zod → persiste `claude_request_logs`.
3. **LLM judge** (se `options.judge`): Haiku confere estrutura → refaz a análise **1x** se reprovar. Judge conta no budget, **não** consome créditos.
4. **consume on-success** (se `ctx.credits`): `consume(ws, userId, cost, ctx)` no fim (§4.11). Falha → nada deduzido.

Erro esperado = shape discriminado `{ ok:false, error:{ code, message } }` (Regra 7): `BUDGET_EXCEEDED` | `INSUFFICIENT_CREDITS` | `VALIDATION` | `API_ERROR`.

## Dois gates de custo (não confundir)

- **Budget hard-cap (Regra 20)** — protege a MARGEM da Criation. Soma USD do mês vs budget BRL do plano. SEMPRE roda (inclui judge).
- **Créditos (creditService)** — cobra o USUÁRIO. Só com `ctx.credits`. Pré-flight 1.5x + consume on-success.
