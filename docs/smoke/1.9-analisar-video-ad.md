# Smoke — 1.9 `analisar.video_ad` (Quick) + `/estudio/analisar/nova`

Primeiro pipeline real do Estúdio fim-a-fim. Liga UI → action → Trigger.dev →
claude.service → créditos → resultado.

## Pré-requisitos

- `pnpm dev` rodando.
- `pnpm trigger:dev` rodando (a task `estudio-analisar-video-ad` executa local).
- `ANTHROPIC_API_KEY` em `.env.local` (validada com chamada real Haiku na 1.8).
- Workspace com ≥1 campanha Meta conectada + criativos sincronizados (Vinicius:
  917 campaigns / 32 ad accounts).
- Saldo de créditos > 0 (signup_bonus da 1.5 cobre).

## Cenários

### 1. Fluxo feliz (campanha conectada)

- [ ] `/estudio/analisar/nova` carrega; tab "Anúncio em vídeo" ativa, demais "em breve".
- [ ] Select de campanha lista campanhas ACTIVE reais.
- [ ] Ao escolher campanha, o select de criativo popula (server action `getCreativesForPicker`).
- [ ] Rodapé mostra "Custo: 1 crédito · ~30s · saldo: N".
- [ ] "Executar análise" → redireciona pra `/estudio/analisar/[id]` em estado **Executando**.
- [ ] Após ~30s, refresh da página mostra **Concluída** com: verdict (Forte/Mediano/Fraco),
      score /100, summary, maior gargalo, pontos fortes/fracos, recomendações.

### 2. Persistência (DB)

- [ ] `analyses`: status='completed', credits_consumed=1, credit_transaction_id preenchido,
      trigger_job_id preenchido, started_at/completed_at.
- [ ] `analysis_results`: 1 row (result_data válido contra analysisQuickOutputSchema, model_used,
      input/output tokens, processing_time_ms).
- [ ] `credit_transactions`: 1 row negativa (consume), idempotencyKey = analysisId.
- [ ] `claude_request_logs`: ≥1 row (a análise; +1 se judge rodou) com cost_usd.

### 3. Gates

- [ ] Saldo insuficiente: action retorna erro inline "Créditos insuficientes" **sem** criar row
      nem disparar job.
- [ ] Budget hard-cap (Regra 20): se workspace estourou o teto Claude do plano, a task marca
      failed com BUDGET_EXCEEDED (sem cobrar crédito — consume só on-success).

### 4. Idempotência / erros

- [ ] Reexecutar a MESMA analysisId (retry da task) não cobra em dobro (idempotencyKey=analysisId).
- [ ] Output inválido do Claude → status failed com VALIDATION (não persiste result).
- [ ] Histórico `/estudio/analisar` lista as últimas 20 com badge de status correto.

## Gotchas conhecidos

- **`ANTHROPIC_API_KEY` no Trigger cloud**: adicionada ao `TRIGGER_RUNTIME_ENV_KEYS` em
  `trigger.config.ts`. Sem `pnpm trigger:deploy` após esta mudança, a task quebra em prod com
  "ANTHROPIC_API_KEY não configurada". Validar com run real no cloud.
- **Métricas de vídeo nulas**: `funnelMetrics.hookRate`/`holdRate` saem null — Meta v25
  descontinuou `video_*_sec_watched_actions`. O prompt trata null sem inventar.
- **Quick MVP = métricas + copy**: sem transcrição (Deepgram) nem frames. Esses entram em Deep.
- **Polling**: nesta sessão o estado running exige refresh manual. Polling automático = 1.10;
  relatório rico de 8 seções = 1.11.
