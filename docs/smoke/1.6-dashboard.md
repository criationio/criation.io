# Smoke E2E manual — Sessão 1.6 Dashboard

**Última atualização:** 2026-05-28
**Status:** Documento de checklist — executar quando suspeitar de regressão visual ou após mudança grande no dashboard.

## Pré-requisitos

- Dev server rodando em `http://localhost:3001` via `pnpm dev`
- Usuário logado em workspace com `onboarding_step = 'completed'`
- Tema dark ativo (padrão) — repetir flow no light se mudou design tokens
- DevTools aberto (Console + Network) pra detectar erros silenciosos

## ✅ Checklist por área

### 1. Boot do dashboard

- [ ] `GET /dashboard` retorna 200 sem flash de erro
- [ ] Loading skeleton aparece brevemente em throttle "Slow 3G" (Network tab)
- [ ] Sem hydration mismatch warnings no Console
- [ ] Sem erros TypeScript do React (especialmente nos widgets recharts)

### 2. KPI Cards (6 cards)

- [ ] Faturamento, Lucro, ROAS, Investimento, CAC, Ticket médio renderizam
- [ ] Cada card mostra: label uppercase, número grande tabular, delta % com seta, sparkline 14d
- [ ] Sparklines têm gradient fade (não linhas chapadas)
- [ ] CAC com `invertDeltaPolarity`: delta negativo aparece em verde (CAC caiu = bom)
- [ ] Hover em qualquer card mostra ícone `?` no header (tooltip)
- [ ] Badge "exemplo" no canto superior direito quando workspace sem dados
- [ ] Quando workspace TEM dados reais mas KPI for 0: mostra `—` em vez de `R$ 0,00`

### 3. Funil de conversão (FunnelPyramid)

- [ ] Forma piramidal real (topo largo, base estreita) — visualmente reconhecível
- [ ] 8 etapas por default: Impressões → Cliques → PageView → Lead → Checkout → Aprovadas → Confirmado → Subs ativas
- [ ] Drop-off pills nomeadas entre etapas: `↓ CTR 2.4%`, `↓ Page Load 87%`, `↓ Lead Rate 34%`, etc.
- [ ] Pills com cor por severidade: < 20% vermelho, 20-50% amarelo, > 50% neutro
- [ ] Click em qualquer etapa expande card de detalhe abaixo
- [ ] Botão **Configurar** abre menu dropdown
- [ ] Preset VSL direto: esconde "Leads"
- [ ] Preset Webinar: mostra "Leads" + "PageView"
- [ ] Preset WhatsApp: esconde "PageView" e "Checkout"
- [ ] Toggle individual de etapa funciona; mudar manualmente vira "Custom"
- [ ] Setting persiste localStorage entre refreshes (`criation_funnel_preset_v1`)

### 4. SalesVsInvestmentChart

- [ ] 3 séries renderizam: Receita (área verde), Investimento (linha amarela), Lucro (linha roxa)
- [ ] Eixo X mostra datas formatadas pt-BR (ex: "15 jun")
- [ ] Eixo Y formato compact BRL (ex: "R$ 4,5k")
- [ ] Hover mostra tooltip custom com data + 3 valores BRL formatados
- [ ] Legend embaixo com 3 cores

### 5. Top Criativos Table

- [ ] 10 criativos renderizados com thumbnail placeholder + nome
- [ ] Colunas: Criativo, Spend, Receita, ROAS, CTR, Conversões, Status
- [ ] Sort: clica no header → toggle asc/desc; primeira coluna não sortable
- [ ] Status pills: "Escalando" (verde), "Maduro" (roxo), "Fadigado" (vermelho), "Testando" (amarelo)
- [ ] ROAS formatado como `2.50×`
- [ ] CTR formatado como `2,4%`

### 6. Channel Mix Donut

- [ ] Donut com hole no centro mostrando "Total R$ X"
- [ ] Slices: Meta, Google, Orgânico, Direct, Email, WhatsApp (cores próprias)
- [ ] Hover em slice destaca + tooltip BRL
- [ ] Legend lateral lista cada canal com %, BRL e cor matching
- [ ] Quando real data retorna vazio → empty state "Sem atribuição ainda"

### 7. UTM Source Table

- [ ] Lista source/medium agrupado
- [ ] ROAS colorido: > 2× verde, < 1× vermelho, 1-2× amarelo
- [ ] Sort funciona (revenue/orders/spend/roas)
- [ ] "—" quando spend = 0 (organic/direct)

### 8. Cohort Heatmap

- [ ] 6 cohorts (mês-de-aquisição) com 8 colunas (M+0 a M+7)
- [ ] Color intensity proporcional ao LTV (gradient accent)
- [ ] Cohorts antigos têm todas cells; recentes têm cells null (dashed)
- [ ] Hover em cell mostra tooltip com cohort + month + valor BRL
- [ ] Legend embaixo com scale 0 → max
- [ ] Sem dados (< 5 vendas/cohort) → empty state "Sem dados suficientes"

### 9. Filtros (DashboardFiltersBar sticky)

- [ ] Filter bar sticky no topo durante scroll
- [ ] Período: dropdown abre, seleção atualiza URL (`?period=last_7d`), valores nos widgets re-renderizam
- [ ] Comparação: "Sem comparação" esconde delta % nos KPI cards
- [ ] Atribuição: só `Last-click` clicável; demais com badge "em breve"
- [ ] Canais: multi-select (checkboxes); URL agrega `?channels=meta,google`
- [ ] Produtos: multi-select; mostra 5 mockados quando workspace sem `gateway_products`
- [ ] Funis: dropdown lista funis salvos + "+ Gerenciar funis" link
- [ ] Botão "Resetar" limpa todos filters preservando ?view
- [ ] Chip "Atualizando…" pisca brevemente durante transition

### 10. Saved Views

- [ ] Dropdown no header lista views existentes
- [ ] Bootstrap: primeira visita cria view "Geral" default
- [ ] "Salvar layout atual como nova view" abre modal
- [ ] Modal: nome + toggle shared + toggle default
- [ ] Salvar → URL muda pra `?view=<id>`, dropdown atualizado
- [ ] Per item: ícone Star (default) + ícone Trash (excluir)
- [ ] Trocar view → layout muda imediato (key={viewId} faz remount)

### 11. Drag-drop layout (DashboardGrid)

- [ ] Botão "Editar layout" no canto superior direito
- [ ] Modo edit: handle grip (⋮⋮) visível em cada widget
- [ ] Arrasta widget pelo corpo → outros se reorganizam
- [ ] Resize pelo canto inferior-direito funciona
- [ ] Botões/inputs dentro de widgets NÃO disparam drag (draggableCancel)
- [ ] Chip "Salvando…" pisca no header durante debounce (1s)
- [ ] "Resetar layout" volta pro DEFAULT_LAYOUT
- [ ] Lock toggle desativa drag/resize

### 12. /configuracoes/funis

- [ ] Lista funis existentes em tabela
- [ ] "Novo funil" abre modal com: nome, descrição, UTM pattern, landing pattern, multi-select produtos, toggle default
- [ ] Salvar → funil aparece na tabela + no dropdown do dashboard
- [ ] Editar funil funciona (modal pré-preenchido)
- [ ] Excluir funil → confirm dialog → soft delete
- [ ] Setar default → estrela move
- [ ] Empty state quando workspace sem funis

## 🐛 Bugs comuns a observar

| Bug                     | Onde verificar                                                               |
| ----------------------- | ---------------------------------------------------------------------------- |
| Hydration mismatch      | Console errors; especialmente FunnelPyramid (localStorage) e KpiCard (useId) |
| Filtros não propagam    | Mudar período → KPI cards atualizam? URL persiste?                           |
| Drag não dispara        | Modo edit ativo? React 19 + react-grid-layout v2 deve estar ok (não v1)      |
| Skeleton trava          | Network throttle pra "Slow 3G" — deve render skeleton inicial                |
| LocalStorage corrompido | DevTools → Application → clear `criation_*` keys                             |
| Theme light vs dark     | Trocar tema via /configuracoes — popover/glass devem adaptar                 |

## 📊 Cobertura automatizada vs manual

| Camada                                                                 | Cobertura                              |
| ---------------------------------------------------------------------- | -------------------------------------- |
| Lógica pura (channel-mapping, period-range, mock-data, funnel-presets) | ✅ Vitest unit tests (PR-15)           |
| Server actions (dashboard-layouts, dashboard-funnels)                  | ⚠️ DB-bound; integration test deferido |
| Server queries (dashboard-metrics)                                     | ⚠️ DB-bound; integration test deferido |
| Componentes React                                                      | ⚠️ Visual + smoke manual (este doc)    |
| Drag-drop interaction                                                  | ⚠️ Smoke manual obrigatório            |
| Hydration SSR/CSR                                                      | ⚠️ Smoke manual obrigatório            |
