## Detalhes por dependência

### 1. OAuth verification Google (timeline 2-6 semanas)

**O quê:** Submeter o app OAuth Client (`566097966780-4bn934o29n807sgva6u2i65m2ud048s0.apps.googleusercontent.com`) pra Google review.

**Pré-req:**

- Dependência 3 (`criation.io` verificado no Search Console).
- `https://criation.io/privacy` + `https://criation.io/terms` páginas reais publicadas (não placeholders).
- App em estado "ready for verification" no GCP Console → APIs & Services → OAuth consent screen.
- Demo video do app gravado mostrando os 3 sensitive scopes em uso (`auth/datamanager`, `auth/adwords`, `auth/cloud-platform`).
- Justificativa escrita de por que cada scope é necessário (Data Manager API canonical conforme ADR-015).

**Onde:**

1. GCP Console → APIs & Services → OAuth consent screen → botão "Publish App" (sai de Testing pra "In production")
2. Aparece "Prepare for verification" → preencher form com justificativas + upload video → submit

**O que destrava:** Qualquer Google account pode autorizar (não só os 100 test users).

**Risco:** Google pode pedir revisões (loop adicional ~2 semanas cada). Comum em primeira tentativa.

### 2. Developer Token Basic access (timeline 1-2 dias úteis)

**O quê:** Solicitar upgrade do developer token (`CC5xew0aCGNM63rfQtM9lA`) de "Test access" pra "Basic access" no API Center da MCC Google Ads.

**Onde:**

1. https://ads.google.com → login com `me@heywhispa.com` (owner da MCC)
2. Menu Tools & Settings (chave inglesa) → Setup → API Center
3. Botão "Apply for Basic access"

**O que destrava:** Queries Google Ads API REST (metadata: `listAccessibleCustomers`, `getCustomerSelf`, etc.) contra contas de produção. Hoje Test access só funciona contra test accounts criadas dentro da MCC.

**Lembrar:** Data Manager API (`events:ingest`) **não usa** developer token, então fanout funciona sem Basic. Mas wizard `/configuracoes/google/conversoes` populando lista de contas + conversion actions precisa.

### 3. Domínio `criation.io` no Search Console (timeline ~5 min + DNS TTL)

**O quê:** Provar pra Google ownership de `criation.io`.

**Onde:**

1. https://search.google.com/search-console
2. Add Property → Domain → `criation.io`
3. Adiciona TXT record no DNS (registrador atual)
4. Espera DNS propagar (~10-60 min)
5. Verify

**O que destrava:** Adicionar `criation.io` como **Authorized domain** no OAuth consent screen. Sem isso, item 1 (OAuth verification) não pode submeter.

### 4. Deploy app em Vercel `gru1`

**O quê:** Deploy de produção do app criation-io em Vercel (`gru1` = São Paulo per CLAUDE.md).

**Pré-req:**

- Branch `main` apontada como production em Vercel
- DNS `criation.io` apontando pra Vercel (CNAME ou A)
- Env vars de produção configuradas no painel Vercel (matching Trigger.dev prod env — `DATABASE_URL`, `SUPABASE_*`, `ENCRYPTION_KEY*`, `META_*`, `GOOGLE_*`, etc.)
- Adicionar `https://app.criation.io/api/oauth/google/callback` aos Authorized redirect URIs do OAuth Client no GCP

**O que destrava:**

- Script `criation-tracking.js` servido de domínio real (não localhost/tunnel)
- Webhooks Hotmart/Kiwify/Eduzz apontam pra prod
- Cliente alpha pode plugar conexões

### 5. Cliente alpha com Meta + Google Ads ativos

**O quê:** 1 usuário (interno AGC ou amigo) com:

- Landing page real instrumentada com `criation-tracking.js`
- 1 conexão Hotmart **prod** ativa (opcional Kiwify/Eduzz pra variar gateway)
- Meta Pixel + ads ativos R$200-500 budget apontando pra landing
- Google Ads + conversion actions ativos R$200-500 budget apontando pra landing
- Consent banner mínimo no site pra Consent Mode v2 read funcionar

**O que destrava:** Dado real fluindo pelas 5 camadas (captura → matching → atribuição → fanout Meta → fanout Google).

**Recomendação:** Site AGC primeiro (zero risco contratual). Se inviável, amigo com landing existente — alinhar expectativa "não posso garantir 100% precisão por 7d, é validação."
